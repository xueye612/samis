# 麻醉记录单持久化与设备模拟验收说明

## 1. 刷新不丢数据

1. 打开麻醉记录单，添加事件、用药、液体、生命体征
2. 切换页码到第 2 页（如有多页）
3. F5 刷新
4. 预期：趋势图、用药线、事件、页码均恢复

## 2. 检查 IndexedDB

1. Chrome DevTools → Application → IndexedDB → `samis_anesthesia_local_v1`
2. 查看 `records` 表 `case_payload`
3. 查看 `medications` / `vital_signs` / `timeline_events` 子表

## 3. 本地队列

1. 修改记录后顶部显示「待上传 N」
2. Application → `sync_queue` 出现 pending 行
3. 刷新后 pending 仍在

## 4. mock 同步

1. 保持在线，等待 ~5–15s
2. 顶部出现「同步成功 HH:mm:ss」
3. sync_queue 对应行 status=success
4. Network 无真实请求（mock 内存路由）

## 5. 失败重试

1. DevTools → Network → Offline
2. 继续编辑，pending 增加
3. 恢复 Online，pending 下降

## 6. 监护仪模拟

1. 点击「启动监护仪模拟」
2. 3–5s 内趋势图新增点；`monitor_raw` 有数据
3. 「停止监护仪模拟」后不再新增

## 7. 呼吸机模拟

1. 点击「启动呼吸机模拟」
2. ETCO2/RR 相关曲线更新；`ventilator_raw` 有数据

## 8. 打印结构

1. 打印预览布局与改动前一致
2. 锁定/打印流程仍可用

## 9. 刷新不抖动

1. 打开已有本地数据的麻醉记录单
2. F5 刷新
3. 预期：先见固定高度 loading 骨架，完成后一次呈现记录单；无明显「先 mock 再本地」闪变
4. 页码与趋势图稳定，不重复闪烁初始化

## 10. 状态提示不遮挡记录单

1. 观察顶部 32px 状态栏（`RecordStatusBar`）：待上传、冲突、网络、最近同步
2. 修改记录、产生 pending/冲突、启动设备时，记录单打印区域宽高不变
3. 右侧「设备与同步」默认紧凑（约 160px 核心区），不挤压下方方案/麻醉方式面板
4. 进入打印预览或 `@media print`：状态栏、设备面板、同步/冲突入口隐藏

## 11. 设备模拟不修改患者信息

1. 记录患者姓名、性别、年龄、诊断、手术名称
2. 启动监护仪模拟，等待若干采集周期
3. 预期：上述字段不变；`monitor_raw` / `vital_signs` 有新增；趋势图更新
4. 启动/停止呼吸机同理，仅 ETCO2/RR/气道压等相关点变化
5. IndexedDB `snapshots` 与 `case_payload` 中 patient/operation 字段未被设备 tick 改写

## 12. 未打开记录单禁止设备模拟

1. 未选择病例或记录缺少 operation_id/patient_id 时点击「启动监护仪模拟」
2. 预期：提示「请先打开或创建麻醉记录单后再启动设备模拟」，设备不启动

## 13. 本地清理策略

1. 单元测试：`anesthesiaLocalCleanupService.test.ts`
2. 预期：`pending` / `failed` 队列仍在；未同步 raw 仍在
3. 非当前记录、已同步、超过 7 天 raw 可被清理
4. 超过 3 天的 success 队列可清理（当前记录最近 200 条 success 保留）

## 14. Playwright E2E

```bash
npm run test:e2e -- e2e/anesthesia-persistence.spec.ts
npm run test:e2e -- e2e/anesthesia-device-mock.spec.ts
npm run test:e2e -- e2e/anesthesia-print-layout.spec.ts
```

覆盖：刷新恢复、设备不改患者信息、状态栏不挤压打印区、打印媒体隐藏状态栏。

## 16. 设备模拟三模式

1. 打开右侧「设备与同步」
2. 切换：正常模拟 / 异常模拟 / 抢救模式
3. 正常：vitals 稳定，5 分钟/条，raw 4s
4. 异常：勾选异常类型，启动监护仪，右侧面板在连续异常后出现提醒
5. 抢救：显示「抢救采集中」，1 分钟/条，raw 2s，趋势图无同 x 堆叠

## 17. 跨页 vital 验收

1. 长时段病例（或多页）
2. 页边界附近 vital：当前页 `[start, end)`，末页含 end
3. 切换页码：边界点只出现在一页
4. 单元测试：`recordPaginationEngine.test.ts`

## 18. 冲突 E2E 验收

1. **开发模式 + mock**（`import.meta.env.DEV && VITE_ANESTHESIA_USE_MOCK`）下可见「注入测试冲突」
2. 点击「注入测试冲突」（或使用 `window.__samisAnesthesiaE2E.injectConflict`）
3. 顶部「冲突 N」→ 打开面板 → 查看 local/server payload
4. 「使用服务器版本」或「重试同步」后冲突数减少

## 19. 打印态验收

1. 更多 → 打印预览
2. 不显示：状态栏、设备面板、注入测试按钮、picker 选择按钮、蓝色虚线
3. 记录单主体完整

## 20. 待完善列表
