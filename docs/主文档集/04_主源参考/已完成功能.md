# 当前已完成功能清单

> 统计口径：基于当前仓库代码（后端路由、前端页面、移动端页面）整理，反映“已落地可开发联调”的范围。  
> 更新时间：2026-05-07

## 1. 后端（backend-tp6）

### 1.1 鉴权与基础能力

- 健康检查：`GET /api/health`
- 登录登出、用户上下文：
  - `POST /api/auth/login`
  - `POST /api/auth/logout`
  - `GET /api/auth/profile`
  - `GET /api/auth/menu`
  - `GET /api/auth/permission`
- 中间件已接入：
  - JWT 鉴权（`JwtAuth`）
  - 审计日志（`AuditLog`）

### 1.2 系统管理（只读查询能力已落地）

- 用户列表/详情：`/api/system/user/list`、`/api/system/user/detail`
- 角色列表：`/api/system/role/list`
- 权限列表：`/api/system/permission/list`
- 字典类型/条目：`/api/system/dict/types`、`/api/system/dict/items`

### 1.3 手术主数据与排台

- 手术主记录：今日、列表、详情、新增、更新、取消、中止
- 排台：列表、保存、批量保存、换房、换序
- 人员安排：列表、保存、批量保存、删除
- 状态日志：列表
- 关键时间：详情、保存

### 1.4 麻醉记录主线

- 记录单：列表、详情、初始化、保存、签名、撤签、锁定、归档
- 体征：批量保存、时间范围查询
- 用药：列表、批量保存、删除
- 事件：列表、批量保存、删除
- 抢救模式：列表、开始、结束、新增抢救事件

### 1.5 PACU 恢复室

- 主记录：列表、详情、初始化、更新基础信息、入室、观察、出室、转运
- 体征：批量保存、范围查询
- Aldrete 评估：列表、保存
- 用药：列表、保存、删除
- 入出量：列表、保存、删除

### 1.6 移动端聚合接口

- 首页聚合：`GET /api/mobile/home`
- 手术列表：`GET /api/mobile/surgery/list`
- 手术详情：`GET /api/mobile/surgery/detail`

## 2. PC 管理端（frontend-admin）

### 2.1 已落地页面（有独立业务页面）

- 登录页：`/login`
- 工作台：`/dashboard/home`
- 手术管理：
  - 手术列表/今日手术
  - 排台管理
  - 急诊登记
  - 手术详情抽屉、手术信息编辑、人员编辑
- 麻醉工作站：
  - 麻醉记录列表
  - 麻醉记录详情（体征/用药/事件/签名/状态流转）
- PACU：
  - PACU 列表
  - PACU 详情
- 异常页：`/403`、`/404`

### 2.2 已接入机制

- 登录态与菜单动态路由注入（基于 `/api/auth/menu`）
- 路由级权限校验（`meta.permission`）
- API 模块化：`auth.ts`、`surgery.ts`、`anes.ts`、`rescue.ts`、`pacu.ts`

### 2.3 当前仍为占位/二期扩展入口

- `dashboard/todo`、`dashboard/sign` 为占位页
- 后端菜单中未对应到视图组件的路由，会自动回退到 `PlaceholderView`
