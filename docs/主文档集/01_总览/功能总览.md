# 手术麻醉系统功能总览（按业务模块）

生成时间：2026-06-02
依据来源：
- 主源：docs/02_specs/、docs/03_dictionary/、docs/04_delivery/
- 补充：samisweb 原型、麻醉记录单 FEATURES、mazuixitong 功能描述
- 实现核对：backend-tp6 migrations / frontend-admin 实际页面 / samisweb mock 实现 / 麻醉记录单 Vue 组件

字段含义：
- 实现状态：已落地 / 部分落地 / 仅原型 / 仅规划
- 是否需要后续开发：是 / 否 / 仅完善
- 是否可复用护理：可直接复用 / 在护理基础扩展 / 麻醉独立建设

---

## 1. 工作台

- 模块作用：登录后首页，呈现今日手术、待办、状态总览。
- 主要页面：今日概览、我的待办、待签名病历、手术间状态总览。
- 核心功能点：手术日程卡片、PACU 状态、质控预警、跳转入口。
- 实现状态：部分落地（PC HomeView 已接真实接口；samisweb 工作台原型最完整）。
- 是否需要后续开发：是（完善质控预警与待办的接口对接）。
- 复用边界：在护理基础扩展（沿用护理登录 / 菜单 / 人员）。

## 2. 手术排班 / 患者列表

- 模块作用：接收手术申请、维护排台、人员与房间安排。
- 主要页面：手术申请接收、手术排台管理、手术房间安排、急诊手术登记、手术详情抽屉、人员编辑。
- 核心功能点：今日列表、列表分页、新增、更新、取消、中止、批量保存、换房换序。
- 实现状态：已落地（PC frontend-admin + backend-tp6 surg_* 表 + /api/surgery/*）。
- 是否需要后续开发：仅完善（与护理 operatenotice 的字段映射闭环）。
- 复用边界：在护理基础扩展（surg_* 表与护理 operatenotice 字段对齐）。

## 3. 术前访视

- 模块作用：麻醉医生术前访视、麻醉评估、ASA 分级。
- 主要页面：麻醉前访视、术前评估表、签名提交。
- 核心功能点：病史、过敏、气道评估、ASA、风险条目、签名、状态流转。
- 实现状态：部分落地（后端有 anes_pre_visit / anes_visit_item，PC 页面占位为主；samisweb 有原型表单）。
- 是否需要后续开发：是（完善表单 + 列表 + 详情联调）。
- 复用边界：麻醉独立建设（与护理 before_operation_visit 字段独立，不复用其表）。

## 4. 麻醉计划

- 模块作用：术前确定麻醉方式、用药、监护与气道方案。
- 主要页面：麻醉计划页（通常与术前评估同屏或子页）。
- 核心功能点：麻醉方式选择、计划用药、监护项、特殊体位、困难气道预案。
- 实现状态：仅原型（samisweb AnesthesiaPlan / 麻醉记录单内含计划字段）。
- 是否需要后续开发：是（落到 anes_operation_ext 与 anes_pre_visit）。
- 复用边界：麻醉独立建设。

## 5. 麻醉记录单

- 模块作用：术中核心电子文书，覆盖时间轴、体征、用药、事件、签名。
- 主要页面：麻醉记录主单（纸质版 + 实时版）、9 个 Tab、关键时间线。
- 核心功能点：
  - 13 个关键时间节点、4 类人员
  - 8 项生命体征异常阈值与闭环
  - 用药 / 输液 / 输血时间段（含双人核对）
  - 完整性检查（16+ 条规则）
  - 电子签名、只读锁定、解锁修改留痕
  - 操作日志、修改前后值留痕
  - 打印预览（A4 适配）
- 实现状态：已落地（麻醉记录单专项 100%；后端 anes_record / vital / medication / event / signature 已建；PC frontend-admin 详情页已接真实接口）。
- 是否需要后续开发：仅完善（把记录单专项整合进统一前端壳，并接真实后端）。
- 复用边界：麻醉独立建设（护理无对应文书）。

## 6. 术中事件与用药

- 模块作用：术中事件标记、麻醉用药、抢救事件、特殊事件。
- 主要页面：用药弹窗、事件弹窗、抢救面板、麻精处方。
- 核心功能点：事件类型字典、用药执行人 / 核对人、高警示药品、抢救模式开关。
- 实现状态：已落地（后端 anes_event / anes_medication / 抢救模式 anes_rescue_session）。
- 是否需要后续开发：仅完善（事件字典与麻精处方流程）。
- 复用边界：麻醉独立建设；麻醉药特殊管制可参考护理 narcotics domain（不直接共表）。

## 7. 生命体征与设备数据

- 模块作用：术中生命体征记录、设备采集接入。
- 主要页面：实时趋势图、监测项录入、设备采集面板。
- 核心功能点：5 分钟 / 1 分钟频率、抢救自动调频、异常识别、设备状态、采集日志。
- 实现状态：部分落地（手工录入与异常识别已完成；设备采集为占位）。
- 是否需要后续开发：是（设备采集为二期重点）。
- 复用边界：麻醉独立建设。

## 8. 液体 / 输血 / 出入量

- 模块作用：术中液体管理、输血记录、出入量平衡。
- 主要页面：输液输血弹窗、出入量录入、出入量平衡卡。
- 核心功能点：晶体 / 胶体 / 血制品、输血双人核对、出量 4 类、平衡计算。
- 实现状态：已落地（记录单专项已完成；后端字段在 anes_record 中，输血聚合可读护理 blood 数据）。
- 是否需要后续开发：仅完善（与护理输血记录的引用对照）。
- 复用边界：可复用护理输血主表（HuliBloodGateway 只读），麻醉端字段独立。

## 9. PACU

- 模块作用：术后恢复室管理。
- 主要页面：PACU 患者列表、入出恢复室、恢复室体征、Aldrete 评估、用药、计费。
- 核心功能点：状态机（waiting_in → recovering → reach_discharge → discharged → completed）、Aldrete 内置、转运、交接。
- 实现状态：已落地（后端 PACU 主线 API + Aldrete + 体征 + 用药 + 入出量；前端 PacuListView / PacuDetailView 已接真实接口）。
- 是否需要后续开发：仅完善（计费与转运细节）。
- 复用边界：麻醉独立建设（护理交接表仅作只读对照）。

## 10. 术后随访

- 模块作用：术后访视、镇痛、并发症、满意度。
- 主要页面：术后随访、术后镇痛、并发症录入、不良事件。
- 核心功能点：随访表单、镇痛 NRS / VAS、恶心呕吐、并发症登记。
- 实现状态：部分落地（后端 anes_post_visit 已建；前端为原型）。
- 是否需要后续开发：是（完善随访表单与列表）。
- 复用边界：麻醉独立建设（与护理 after_operation_visit 字段独立）。

## 11. 麻醉小结

- 模块作用：手术后形成的麻醉过程总结，作为文书归档。
- 主要页面：麻醉小结页面。
- 核心功能点：过程文本、术后情况、评价等级、与记录单数据联动。
- 实现状态：部分落地（前端骨架，后端走文书状态机）。
- 是否需要后续开发：是（落地汇总展示、签名与归档）。
- 复用边界：麻醉独立建设。

## 12. 质控中心

- 模块作用：麻醉专业医疗质量控制。
- 主要页面：质控总览、26 项指标、低体温 / PACU 专项、缺陷列表、PDCA。
- 核心功能点：
  - 一期：基础工作量、运营指标
  - 二期：26 项国家质控指标（结构 3 / 过程 6 / 结果 7 / PACU 术后 8 / 产科 1，与原型一致）
  - 指标分子分母穿透、缺陷识别、整改闭环
- 实现状态：仅原型（samisweb 完整原型；后端尚未建 qc_* 表）。
- 是否需要后续开发：是（二期重点）。
- 复用边界：麻醉独立建设。

## 13. 统计报表

- 模块作用：工作量、麻醉方式、运营指标统计与导出。
- 主要页面：工作量统计、麻醉方式分布、运营指标。
- 核心功能点：时间范围筛选、卡片指标、趋势图、导出 Excel / PDF。
- 实现状态：部分落地（一期基础统计已在质控规范内）。
- 是否需要后续开发：是（与质控共用数据底座）。
- 复用边界：可复用护理 statistics domain 部分能力。

## 14. 系统管理

- 模块作用：用户、角色、权限、菜单、字典、设备、接口配置、审计日志。
- 主要页面：用户管理、角色管理、权限管理、菜单管理、字典管理、审计日志。
- 核心功能点：列表 / 详情查询（只读已落地）、维护类（一期受限）。
- 实现状态：部分落地（只读查询接口已开放）。
- 是否需要后续开发：仅完善（维护类接口与页面）。
- 复用边界：可直接复用护理体系（sys_user / sys_role / sys_permission / sys_menu / sys_dict_* / sys_audit_log 已在 Phase 1 与护理对齐）。

## 15. 数据字典

- 模块作用：业务字典维护（麻醉方式、事件类型、药品、液体、血制品、模板）。
- 主要页面：字典类型、字典条目、麻醉专用字典。
- 核心功能点：分类管理、条目维护、状态启用 / 禁用。
- 实现状态：部分落地（基础字典已通；麻醉专用字典与模板需补）。
- 是否需要后续开发：是（麻醉专用字典 + 打印模板）。
- 复用边界：可直接复用护理字典（sys_dict_*），新增麻醉特有字典分类。

## 16. 权限与日志

- 模块作用：权限点、菜单权限、操作日志、登录日志、接口日志。
- 主要页面：权限列表、菜单管理、审计日志。
- 核心功能点：权限点编码（模块:资源:动作）、菜单与角色绑定、审计中间件。
- 实现状态：已落地（sys_role / sys_permission / sys_menu / sys_audit_log / sys_interface_log + JWT + AuditLog 中间件）。
- 是否需要后续开发：仅完善（页面与导出）。
- 复用边界：可直接复用（与护理体系合流）。

## 17. 接口集成

- 模块作用：与 HIS / EMR / LIS / PACS / 病理 / 设备 / 护理 SoT 的接口。
- 主要页面：接口配置、接口日志、回调入口（后端）。
- 核心功能点：
  - 护理 SoT 只读聚合：/api/nursing/<domain>/list、/detail，domain 含 user/dict/patient/operation/schedule/evaluation/medicant/handover/security
  - 4 个集成网关：HuliBloodGateway / HuliHandoverGateway / HuliOperationGateway / HuliSecurityGateway
  - 外部 HIS/EMR/LIS/PACS 接口回调走 app/controller/Interfaces
- 实现状态：部分落地（护理只读聚合与 4 个网关已实现；外部 HIS/EMR 仍为占位）。
- 是否需要后续开发：是（外部接口随医院环境对接）。
- 复用边界：可直接复用护理 SoT（仅读，不重建主表）。
