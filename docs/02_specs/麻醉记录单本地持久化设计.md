# 麻醉记录单本地持久化设计

## 为什么需要本地持久化

麻醉记录单是基于时间轴的连续记录，包含高频生命体征、用药线段、液体/输血、事件符号等。当前 samisweb 原型在刷新后会丢失运行态数据，无法满足术中连续记录与断网场景。

## 为什么使用 IndexedDB

- 结构化、异步、容量大，适合高频写入与批量查询
- 支持事务，适合「写记录 + 写队列 + 写审计」的原子组合
- localStorage 仅保存轻量偏好（当前病例、页码等），不存完整记录单

## IndexedDB 与 localStorage 分工

| 存储 | 内容 |
|---|---|
| IndexedDB `samis_anesthesia_local_v1` | 完整病例、子表、同步队列、设备 raw |
| localStorage `samis.anesthesiaRecord.localState.v1` | 字典覆盖、UI draft（原有逻辑保留） |
| IndexedDB `settings` | 当前页码、最近打开记录 |

## 本地表结构

数据库：`samis_anesthesia_local_v1`（Dexie，见 `src/services/anesthesia/localDb.ts`）

| 表 | 说明 |
|---|---|
| records | 主记录 + `case_payload` 全量 JSON |
| snapshots | 打印/锁定快照 |
| timeline_events | 事件 |
| medications / fluids / transfusions | 用药、液体、输血 |
| vital_signs | 趋势图/打印显示点 |
| monitor_raw / ventilator_raw | 设备原始数据 |
| io_records / lab_results | 出入量、血气 |
| audit_logs | 本地审计 |
| sync_queue | 待上传队列（仅前端） |
| settings | 页码等偏好 |

## local_id 与 server_id

- 前端生成 `local_id`（如 `case-001`、`vital-173...`）
- 后端返回 `server_id` 后写入对应行与 `records.server_id`
- 幂等键：`operation_id + entity_type + local_id`

## Hydration 状态

store 暴露三个标志（见 `src/stores/anesthesia.ts`）：

| 标志 | 含义 |
|---|---|
| `localPersistenceReady` | IndexedDB 初始化完成 |
| `isHydrating` | 正在从 IndexedDB 读取并合并数据 |
| `hasRestoredLocalData` | 至少合并过一条本地记录 |

流程：

```text
页面进入 / app 启动
  → isHydrating = true
  → 读取 records / settings / sync_queue / conflicts
  → 合并本地 case 覆盖 seed（仅本地有记录时）
  → restoreCasePageNo（页码在记录单渲染前完成）
  → isHydrating = false, localPersistenceReady = true
  → 渲染 LiveAnesthesiaSheet
```

`main.ts` 在 `app.mount()` 前 `await bootstrapAnesthesiaLocalPersistence()`，避免先渲染 seed 再闪变本地数据。

## 刷新恢复逻辑

1. 应用启动：`bootstrapAnesthesiaLocalPersistence()` 从 IndexedDB 合并；**无本地记录时才保留 seed mock**
2. hydration 完成前：`AnesthesiaRecord.vue` 显示 `--record-sheet-skeleton-height` 骨架，不渲染完整记录单主体
3. 打开记录单：`restoreCasePageNo(caseId)` 在 `sheetReady` 为 true 前完成
4. 优先使用本地 `case_payload` 覆盖 seed 中同 id 病例
5. 子表用于同步与设备追溯；UI 主渲染仍来自 `SurgeryCase`

## 页面刷新防抖动

- 不在 hydration 前渲染 `LiveAnesthesiaSheet` 与趋势图
- 顶部状态栏固定高度 32px，chip 出现/消失不改变页面高度
- 设备状态恢复不触发 `syncRecordDocument` 全量重绘
- 切换病例时短暂 `caseSheetReady = false`，避免旧数据闪屏
- 骨架高度使用 CSS 变量 `--record-sheet-skeleton-height`，随 `sheetZoom` 与多页数微调，减少 hydration 完成时高度跳变

## 顶部状态栏

- 组件：`RecordStatusBar.vue`，单行固定 32px
- **仅全局摘要**：待上传（可点击→同步详情弹窗）、冲突（可点击→冲突面板）、失败、同步中、网络在线/离线、最近同步时间
- 本地已保存时间移入「同步详情」弹窗，不在顶部重复
- 成功态弱化（`muted` chip）；失败/冲突突出（红/橙 chip）
- chip 固定高度 22px，出现/消失不改变状态栏 32px 高度
- 打印/打印预览：`@media print` + `.no-print` 隐藏

## 右侧设备与同步面板

- 组件：`RecordDeviceWorkbenchPanel.vue`（折叠项「设备与同步」）
- **仅设备运行摘要 + 详情入口**：监护仪/呼吸机运行状态（行内启动/停止）、最近采集、采样频率只读 chip
- 同步详情 / 冲突入口为文本按钮；失败时显示「同步失败，点击查看」
- 模拟模式、异常类型、可编辑采样间隔折叠在「模拟设置」`<details>` 内
- 核心区域高度约 160px 内，不使用 2×3 大卡片
- 不与顶部重复展示待上传/网络/采集状态/模拟模式摘要
- 打印/打印预览：随 `.record-toolbox` 隐藏

## 开发测试按钮

- 「注入测试冲突」仅在 `import.meta.env.DEV && VITE_ANESTHESIA_USE_MOCK` 时显示
- 生产/演示构建默认隐藏，不出现在临床操作界面

## 全局 bootstrap 门控

- `main.ts`：`app.mount()` 前 `await bootstrapAnesthesiaLocalPersistence()`
- `router.beforeEach`：导航前再次确保 bootstrap 完成（幂等）
- `AppLayout`：`localPersistenceReady && !isHydrating` 前显示轻量 loading，避免业务页误读 seed
- composable：`useAnesthesiaPersistenceGate()` / `ensureAnesthesiaPersistenceReady()`
- **原则**：IndexedDB 有数据 → 合并覆盖 seed；无本地数据 → seed 作为演示 fallback，不作为 hydration 前“真实数据”展示

## 自动保存

- 入口：`store.afterRecordMutation(caseId)` → `schedulePersistCase()` 防抖 1s
- 每次保存：写 records + 子表 + sync_queue（除非 `skipQueue`）
- 页码变更：`livePageNo` watch → `setRecordPageDraft` + 持久化

## 手动保存

「保存草稿」仍写 localStorage UI draft；临床数据通过 IndexedDB 自动保存。

## 锁定/打印

`printAndLockRecord` 触发 `operationType: lock` 入队，快照写入 `snapshots`。

## 清理策略

实现：`src/services/anesthesia/anesthesiaLocalCleanupService.ts`

| 函数 | 作用 |
|---|---|
| `cleanupSyncedDeviceRaw()` | 清理非当前记录、已同步、超过 7 天的 raw |
| `cleanupOldSuccessQueue()` | 清理超过 3 天的 success 队列 |
| `getLocalStorageStats()` | 统计 raw / queue 数量 |
| `runConservativeLocalCleanup()` | 手动触发保守清理 |
| `runStartupLocalCleanupIfDue()` | 启动后每 6 小时最多执行一次 |

保守规则：

- **不清理**：`sync_status !== success` 的 raw、`pending/failed/conflict/uploading` 队列、当前打开记录全部 raw
- **可清理**：非当前记录 + 已同步 + 超过 7 天的 `monitor_raw` / `ventilator_raw`；超过 3 天且不在当前记录最近 200 条内的 success 队列
- 清理结果写入 `audit_logs`（action=`local_cleanup`）

## 性能注意

- 防抖 1s，避免每次按键写库
- 设备 raw 与 vital_signs 分离；趋势图只读 display 点
- 批量上传设备数据，不每条立即请求后端

## 跨页 vital_signs 显示规则

实现：`recordPaginationEngine.isTimeOnPage` + `LiveAnesthesiaSheet.pageVitals` + `selectDisplayVitalsForBand`

| 规则 | 说明 |
|---|---|
| 页内归属 | 非末页：`pageStartTime <= measure_time < pageEndTime` |
| 末页 | `pageStartTime <= measure_time <= pageEndTime`（包含轴终点） |
| 下一页 | `nextStartTime <= measure_time < nextEndTime` |
| 分桶 | 趋势图按 `vitalDisplayIntervalMinutes` 分桶，每桶保留优先级最高的一条 |
| 禁止 | 页外数据不得挤压到页尾；边界点不得在相邻两页重复显示 |

`selectDisplayVitalsForBand` 仅对**当前页已过滤**的 vitals 分桶，抢救模式 1 分钟/条时桶宽为 1，避免同 x 坐标堆叠。

## 空字段提示策略

| 模式 | 纸面行为 |
|---|---|
| 编辑 | picker 类字段 hover 显示选择入口；`PaperFormField` 空值浅色占位，无大面积虚线 |
| 查看 | 只显示文本或空白，不显示选择按钮 |
| 打印 | 隐藏虚线、选择按钮、hover、调试提示 |
| 待完善 | 手术体位、麻醉方法、医师/护士、实施手术、术后去向、麻醉小结等进入右侧「待完善列表」，不在纸面强提示 |

实现：`PaperPickerField` / `AnesthesiaMethodPickerField` / `PaperFormField` 的 `interactionMode`；`buildRecordPendingFields()` → `RecordQualityPanel`。

## 用药子表字段（medications）

`medications` 行除 `payload` 全量 JSON 外，索引字段包括：

| 字段 | 说明 |
|---|---|
| `mode` | `single` / `continuous` / `intermittent`（前端录入为 单次用药/持续泵入/间断追加） |
| `is_special` | 是否进入纸面「辅助及特殊用药」说明区；**不**决定是否画线 |
| `special_no` | 特殊用药区编号（1、2、3…，展示转 ①②③） |
| `event_time` | 单次/间断给药事件时间 |
| `start_time` / `end_time` | 持续泵注区间 |
| `rate` | 泵速等（对应前端 `pumpRate`） |

显示规则详见 [麻醉记录单用药显示规则.md](./麻醉记录单用药显示规则.md)。
