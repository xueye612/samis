# 麻醉设备模拟采集设计

## 设备模拟模式（2026-06-02）

右侧「设备与同步」工作台提供三种模式（`deviceSimulationMode.ts` + `RecordDeviceWorkbenchPanel`）：

| 模式 | raw 频率 | vital_signs 显示频率 | 异常 | 采集状态 |
|---|---|---|---|---|
| 正常模拟 | 4s/条 | 5 分钟/条（可配置 1–5） | 极少，不刷屏 | 采集中 |
| 异常模拟 | 4s/条 | 5 分钟/条 | 可控异常，可选类型 | 采集中 |
| 抢救模式 | 2s/条 | 1 分钟/条 | 可偶发混乱 | 抢救采集中 |

localStorage：

- `samis.anesthesia.deviceSimulationMode`
- `samis.anesthesia.abnormalSimulationTypes`

### 正常模拟

日常演示，稳定范围（`deviceMockSamples.ts`）：

```text
HR: 65–95
SBP: 100–135
DBP: 60–85
SpO2: 96–100
TEMP: 36.0–37.2
EtCO2: 35–45
```

### 异常模拟

- 不进入抢救高频模式
- 每个 vital 显示桶写入一条**选定类型**的异常值
- raw 约 25% 概率带异常，用于追溯
- 可选类型：低血压、高血压、心率过快、血氧下降、EtCO2 异常
- 右侧面板仍遵循「连续/严重才提醒」（`aggregateAbnormalVitalsForPanel`）

### 抢救模式

- 自动进入记录单抢救状态（`enterRescueRecordMode`）
- raw 2s、vital 1 分钟/条
- 趋势图按 1 分钟桶 + 页过滤，避免点位堆叠
- 退出抢救模式：顶部「退出抢救」或切回正常/异常模拟

## 异常模拟（历史）

旧版约 8% 随机异常已废弃，改为上述三种模式独立控制。

## 抢救模式（历史说明）

`rescueMode: true` 时间隔 **1 分钟/条**（覆盖用户配置的 1–5 分钟）

## raw 本地保留策略

见 `anesthesiaLocalCleanupService.ts`：

- 未同步（`local_only` 等）raw **禁止**清理
- 当前打开记录 raw **全部保留**
- 非当前记录、已同步、超过 7 天 raw 可低频清理
- 趋势图只读 `vital_signs`，清理 raw 不影响纸面显示（已同步部分）

## raw 与 display 点

| 表 | 用途 |
|---|---|
| monitor_raw / ventilator_raw | 追溯、批量上传 |
| vital_signs | 趋势图、打印 |

打印只使用 `vital_signs`，不直接使用 raw。

## 上传队列

raw → `entity_type: monitor_raw|ventilator_raw`

vital → `entity_type: vital_sign`

API：

```text
POST /api-samis/pc/v1/anesthesiaDevice/batchPushMonitorData
POST /api-samis/pc/v1/anesthesiaDevice/batchPushVentilatorData
```

## 数据边界（禁止写入患者/手术快照）

设备模拟**只**负责设备相关数据，**不得**写入或修改：

- 患者姓名、性别、年龄、住院号、床号、科室、诊断
- 手术名称、手术通知、麻醉方法、手术人员
- `snapshots`、`operationInfo`、`record snapshot`

允许写入：

| 目标 | 说明 |
|---|---|
| `monitor_raw` / `ventilator_raw` | 设备原始采集 |
| `vital_signs` | 趋势图/打印显示点 |
| `sync_queue` | 待上传队列 |
| `records.case_payload.device` | 最近采集时间、运行状态等元数据 |

患者/手术信息**仅**来源于 `operationInfo/getOperationInfo` 或本地已保存的 `snapshots`。

实现要点：

- `monitorMockService` / `ventilatorMockService` 不调用 `saveCaseToLocalDb` 全量保存
- store 使用 `saveDeviceVitalOnly` / `patchRecordDeviceCollectMeta`，不触发 `syncRecordDocument`
- 设备 tick 只追加 `vitals` 数组，不覆盖完整 `SurgeryCase`

## 启动前校验

启动监护仪/呼吸机模拟前必须满足：

1. 本地持久化已完成（`localPersistenceReady`）
2. 当前已打开麻醉记录（`record_local_id` / `selectedId`）
3. 记录含 `operation_id` 与 `patient_id`
4. 记录未锁定

不满足时提示：**请先打开或创建麻醉记录单后再启动设备模拟**

## 刷新与状态恢复

- 刷新后**不自动启动**监护仪/呼吸机
- 若模拟未运行，状态栏不显示「运行中」
- 可显示最近一次采集时间
- 恢复设备元数据时**不修改**患者信息

## 状态提示位置

- **顶部** `RecordStatusBar`：仅全局同步摘要（待上传、冲突、网络、最近同步），**不**展示设备运行详情
- **右侧**「设备与同步」：监护仪/呼吸机运行状态、最近采集、采样频率 chip；同步/冲突为详情入口
- 采样间隔在主面板为只读 chip（`采样 N 分钟/条`）；可编辑时仅在「模拟设置」内
- 设备启动/停止/采集**不弹大提示**；同步成功仅更新时间 chip
- 打印模式下隐藏状态栏、设备面板、开发测试按钮及所有运行态 UI

## 趋势图刷新

mock 写入 `vital_signs` 后通过 `onDeviceVitalAppended` 轻量更新内存中的 `vitals` 数组，**不**调用 `syncRecordDocument()`。

### 横轴与监测带定位

- 趋势图横轴使用 **含秒的小数分钟**（`timeToFractionalMinutes`），避免设备 3–5s 采集全部落在同一分钟列造成符号堆叠。
- 监测数值带对齐 **5 分钟纸面格**，同一格同一指标只保留 **最新一条** 显示值（`buildMonitorCells` 网格去重）。
- 设备采集时间锚定手术日期（`resolveRecordSheetNowIso`），与记录单时间轴一致。
