# 麻醉系统前端原型优化方向与组件规划

> 本文档面向「方向性优化」阶段，输出整体审查结论、视觉与组件优化方向、数据字典梳理、质控数据字段与统计逻辑，以及可落地的组件规划，便于下一步实际前端开发推进。
>
> **重要约束：本阶段不修改麻醉记录单本体**（`LiveAnesthesiaSheet.vue` 及 `src/components/anesthesia/record/sheet/**`、`AnesthesiaRecord.vue`），仅优化其外围模块、通用组件、数据结构与展示/统计能力。

---

## 一、现有原型整体审查

### 1.1 技术栈与工程现状

- 框架：Vue 3 + TypeScript + Vite；UI 库 Arco Design Vue；图表 ECharts；状态 Pinia；路由 Vue Router。
- 数据：纯前端 Mock + Pinia，`src/services/datasetStore.ts` 充当本地数据集边界，`src/services/mockApi.ts` 作为 API 边界占位。
- 质量：已有 Vitest 单测（16 文件 / 103 用例）与 Playwright 冒烟、截图测试；`scripts/style-guard.mjs` 守卫核心商业化页面禁用硬编码颜色。
- 设计令牌：`src/styles/tokens.css` 已建立较完整的颜色 / 间距 / 圆角 / 字体 / 阴影令牌，并桥接 Arco 主题变量。

### 1.2 已完成模块盘点

| 模块 | 主要页面 | 完成度 | 说明 |
| --- | --- | --- | --- |
| 工作台 | `workbench/*` | 高 | 今日手术、手术间总览、待办、紧急呼叫 |
| 术前管理 | `preoperative/*`、`PreVisit.vue` | 中 | 申请接收、会诊、检查审核、知情同意、安全核查、术前访视 |
| 手术麻醉 | `surgery/*`、`AnesthesiaRecord.vue` | 高 | 排班、记录单（本体，**不动**）、术中用药/输液/事件、麻醉计划/交班/小结 |
| PACU | `pacu/*`、`PacuList/Record/Transfer.vue` | 中-高 | 列表、恢复记录、转出、预警、预约、接收 |
| 术后管理 | `postoperative/*` | 中 | 镇痛、随访、并发症、非计划事件 |
| 麻醉质控 | `quality/*`、`QualityDashboard.vue` | 高 | 26 项指标、缺陷识别、PDCA、专项分析 |
| 统计报表 | `reports/*` | 中 | 工作量、麻醉方式、运营分析 |
| 基础配置 | `config/*` | 中-高 | 手术间、人员、药品/液体/方式/事件字典、评分/打印模板 |
| 系统管理 | `system/*` | 中 | 用户、角色、审计、接口、Mock 配置 |

### 1.3 数据流与交互逻辑

- 全局唯一 store `useAnesthesiaStore`：持有 `cases / pacuPatients / followUps` 等临床数据与全部字典配置；通过 `syncCaseToDataset` 等把临床数据归一化进质控数据集，再由 `qualityCalculator`、`qualityDefectRules` 计算指标与缺陷。
- 字典（药品 / 液体血制品 / 生命体征 / 麻醉方式 / 通用枚举）集中在 `mock/configSeed.ts`，并落地 localStorage（`anesthesiaRecordEngine` 的 `RecordLocalState`）。
- 组件通过字典动态生成内容：用药快捷录入、生命体征图表项、液体录入等均读取 `store.configDrugs / configVitals / configFluids`。

### 1.4 潜在可优化点（外围，不含记录单本体）

1. **页面信息密度与主次不一致**：部分页面（如 `PreVisit.vue`、`AnesthesiaSummary.vue`）为纯表单，缺少汇总卡片、统计与可视化，主次区分弱。
2. **二级模块缺少"概览—明细"分层**：术中用药 / 输液 / 事件页面是全院平铺表格，缺少按患者聚合的汇总视图。
3. **麻醉小结信息不足**：当前仅有效果评价 + 文本域，未体现"用药总览、液体/输血统计、关键事件汇总"，与需求文档不符。
4. **质控数据提取分散**：缺陷规则集中在质控集，但"单病例完整性 / 缺项 / 事件完成率"缺少可复用的病例级提取函数，难以在小结、随访、详情页统一复用。
5. **可复用组件库未成形**：`shared/` 下有 `ModulePageShell / EmptyState / DictCrudPanel` 等，但统计卡片（`MetricCard`）、区块标题、汇总分组等尚未形成统一规范。
6. **视觉细节**：渐变 / 阴影在 hero、header 等处偏多，医疗系统宜更克制；部分页面直接写死 `#64748b` 等十六进制色值，未走令牌。

## 二、视觉与美化优化方向

### 2.1 色彩规范（医疗专业风格）

沿用并收敛 `tokens.css`，明确语义色，避免随意渐变：

| 语义 | 令牌 | 用途 |
| --- | --- | --- |
| 主色（蓝） | `--primary` `#2563eb` | 主操作、选中态、链接 |
| 医疗青/绿 | `--medical-cyan` `--color-success-600` | 正常 / 通过 / 完成 |
| 警告（橙） | `--warning` | 缺项、待办、临界 |
| 危险（红） | `--danger` | 异常值、抢救、未核对 |
| 中性灰阶 | `--color-neutral-*` | 文字层级、边框、背景 |

落地建议：
- **收敛渐变与阴影**：hero/header 的大面积渐变改为 1px 细描边 + 浅底；阴影统一用 `--shadow-xs/-sm`，hover 才升级。
- **杜绝硬编码色值**：将各 `.vue` 中 `#64748b` 等替换为 `var(--text-tertiary)` 等令牌，并逐步把页面纳入 `style-guard.mjs` 守卫列表。

### 2.2 布局与主次

- **核心区突出**：每个业务页采用「页头(标题+操作) → 关键指标卡 → 主内容（表格/表单/图表） → 次要信息（日志/说明）」的纵向节奏。
- **非关键区克制**：说明文字、时间戳、版本信息使用 `--text-tertiary` + `--font-size-xs`。
- **右侧面板规范**：详情/记录类页面右侧固定宽（320–360px）面板承载"质控提示 / 快捷操作 / 最近录入"，与左侧主区形成主次。

### 2.3 自适应与打印

- 栅格统一用 `repeat(auto-fit, minmax(...))` 或断点（已在 `global.css` 的 `@media (max-width: 1200px)` 起步），向下兼容 1366 宽笔记本。
- 关键统计卡 / 表格支持横向滚动（`scroll.x`），避免窄屏挤压。
- 打印：导出/打印预览样式与屏幕样式解耦（`@media print`），保证小结、随访、统计可打印；**记录单本体打印不在本阶段处理**。

### 2.4 可视化建议

| 场景 | 图表 | 要点 |
| --- | --- | --- |
| 生命体征趋势 | 多轴折线（HR/BP/SpO₂/TEMP） | 阈值带 + 异常点高亮，颜色固定（HR 红、BP 蓝、SpO₂ 青、TEMP 橙） |
| 用药结构 | 横向条形 / 分类汇总 | 按药品类别聚合，高警示药品红色标记 |
| 出入量平衡 | 对比条形 / 水位 | 入量 vs 出量，净平衡数值突出 |
| 质控指标趋势 | 折线 + 目标线 | 达标绿、未达标红，鼠标悬浮看分子分母 |
| 关键事件 | 时间轴 | 按阶段分组，严重程度配色 |

## 三、组件与交互优化方向

### 3.1 组件职责划分

| 组件域 | 现有/建议组件 | 职责 |
| --- | --- | --- |
| 患者/手术信息区 | `PatientInfoBar`（建议抽象） | 统一展示患者基本信息 + 手术信息 + 状态标签 |
| 术前访视 | `PreVisit.vue`（重构为分区表单） | 病史 / 过敏史 / 气道评估 / 检查 / 计划 |
| 术后访视 | `postoperative/PostoperativeFollowupPage.vue` | 恢复情况 / 麻醉效果 / 异常事件 |
| 麻醉小结 | `surgery/AnesthesiaSummary.vue`（本次增强） | 用药总览 / 液体输血统计 / 关键事件汇总 / 出入量 / 质控完整性 |
| 麻醉中数据展示 | `surgery/SurgeryMedications/Fluids/Events.vue`、`VitalsChart` | 用药、液体、输血、生命体征、事件 |
| 数据统计与质控 | `quality/*`、`MetricKpiCards`、`IndicatorTrendChart` | 指标、缺陷、趋势 |
| 通用操作面板 | `RecordQuickToolbar`（记录单内，不动）、建议通用 `ActionPanel` | 快捷操作 / 右键菜单 / 批量录入 |

### 3.2 可复用组件库规划（`src/components/shared/`）

建议沉淀以下通用组件，便于后续扩展：

- `ModulePageShell`（已有）：页头 + 工具条 + 统计 + 内容插槽。
- `MetricCard`（已有）：统计卡，含 `variant`（default/warn/danger）。
- `SummaryStatGrid`（建议）：统计卡栅格容器，统一 `repeat(auto-fit,minmax(160px,1fr))`。
- `SectionCard`（建议，可由 `a-card.section-card` 约定替代）：带标题/操作位的内容区块。
- `KeyEventTimeline`（建议）：关键事件时间轴，按阶段/严重度配色。
- `EmptyState`（已有）：空态。
- `DictCrudPanel`（已有）：字典增删改查面板，可被各配置页复用。

### 3.3 交互方向

- **操作路径短**：列表行内直达详情/记录；汇总页一键定位到对应病例。
- **反馈明确**：保存/提交统一用 `Message`；危险操作（解锁、作废）二次确认。
- **批量与快捷**：字典驱动的快捷录入（用药/液体）、右键菜单维护（已在记录单内实现），外围列表逐步对齐。

## 四、数据字典与结构优化方向

### 4.1 现有字典梳理

| 字典 | 来源 | 关键字段 | 备注 |
| --- | --- | --- | --- |
| 药品 `DrugDictItem` | `configSeed.seedDrugDict` | name / category / unit / route / highAlert / defaultDose | 已支持启用开关、高警示标记 |
| 液体血制品 `FluidBloodDictItem` | `configSeed.seedFluidBloodDict` | name / category / subCategory / unit | 区分晶体/胶体/血液制品 |
| 生命体征 `VitalSignDictItem` | `configSeed.seedVitalSignDict` | shortCode / unit / normalRange / chartEnabled / defaultValue | 驱动图表与异常判定 |
| 麻醉方式 `AnesthesiaMethodCategory` | `configSeed.seedMethodCategories` | 分类 + 方式项 | 树形 |
| 通用枚举 | `configGenericDicts` | 血型 / Rh / 输血反应 / 气道 / 插管 / 频率等 | 字符串数组 |
| 事件 / 评分 / 打印模板 | store `configEvents/configScores/configPrintTemplates` | 名称数组 | 建议结构化（见下） |

### 4.2 结构优化建议

1. **字段类型/单位/可选值/默认值**：药品、液体、生命体征已具备，建议把 `configEvents`（事件字典）由字符串数组升级为带 `分类 / 严重度默认 / 是否质控事件 / 阶段` 的结构化对象，便于事件录入与质控统计动态生成。
2. **统一管理术前/术后/小结/术中数据**：均挂在 `SurgeryCase` 上（`preVisit / events / medications / fluids / vitals / outputs`），小结/随访以病例为中心聚合，保证统计与质控一致。
3. **便于统计与质控**：所有可统计数据都应能从 `SurgeryCase` 纯函数派生（见第五节服务），不依赖组件内部状态。

## 五、可提取的质控数据字段与统计逻辑

本阶段新增可复用纯函数服务 `src/services/caseSummaryStats.ts`，从单个 `SurgeryCase` 派生统计与质控完整性，供麻醉小结、术后随访、患者详情等多处复用。

### 5.1 可提取字段

| 维度 | 字段 | 来源 |
| --- | --- | --- |
| 用药 | 总条目、持续/单次数、按类别聚合、高警示数、未核对数 | `case.medications` |
| 液体/输血 | 晶体量、胶体量、血制品量、自体血量、输液总量、血制品袋数、未双核对数、输血反应数 | `case.fluids` |
| 出量 | 尿量、出血量、引流量、合计 | `case.outputs / outputRecords` |
| 平衡 | 入量、出量、净平衡 | 派生 |
| 事件 | 总数、质控事件数、有效事件数、按严重度分布、缺处理措施数 | `case.events` |
| 生命体征极值 | HR/SBP/SpO₂/TEMP 的最小最大、最低体温 | `case.vitals` |
| 时长 | 麻醉时长、手术时长（分钟） | 时间字段派生 |

### 5.2 质控完整性检查（缺项 / 事件完成率）

`caseSummaryStats` 输出 `quality.items`（逐项 ok/warn + 说明）与 `completionRate`：

| 检查项 | 规则 | 缺陷含义 |
| --- | --- | --- |
| 体温监测 | 至少 1 条 `TEMP` 记录 | 缺体温监测（影响体温监测率） |
| 高警示药品核对 | 所有 `highAlert` 用药均有 `checker` | 高警示未双核对 |
| 血制品双签 | 所有血液制品 `doubleCheck=true` | 输血未双核对 |
| 事件处理措施 | 所有有效事件均有 `treatment` | 事件缺处理措施 |
| 事件完成率 | 有处理措施事件 / 有效事件 | 事件闭环度 |
| 出血量记录 | 存在出血量记录 | 出量缺项 |
| 小结签名 | `summary.doctorSigned` | 小结未签名 |

> 这些指标与 `qualityDefectRules` / 26 项指标口径保持一致方向，作为"病例级"快速自检，正式指标仍以质控数据集计算为准。

## 六、功能方向（前端原型可实现）

- **术前访视**：分区表单（基本信息 / 病史过敏 / 气道评估 / 检查记录 / 麻醉计划），BMI 自动计算、ASA 分级、完成度提示。
- **术后访视**：恢复情况、麻醉效果、异常事件勾选与随访建议（`PostoperativeFollowUp` 类型已具备字段）。
- **麻醉小结**（本次增强）：用药总览、液体/输血统计、关键事件时间轴、出入量平衡、质控完整性自检 + 原有效果评价表单。
- **麻醉中数据梳理**：趋势图（`VitalsChart`）、关键事件、入/出量、生命体征极值。
- **质控数据提取**：用药/液体/血制品使用记录、异常值、缺项、事件完成率（`caseSummaryStats`）。
- **打印/导出**：小结、随访、统计的打印预览与 CSV 导出（部分已具备 `exportQualityCsv / exportWorkloadCsv`）。

## 七、下一步落地建议（优先级）

1. **P0**：麻醉小结增强（本次已实现）+ `caseSummaryStats` 服务 + 单测。
2. **P1**：术前/术后访视页重构为分区表单 + 完成度提示；术中用药/液体/事件页增加"按患者聚合"概览。
3. **P1**：事件字典结构化（`configEvents` 升级为对象）。
4. **P2**：视觉收敛（渐变/阴影），硬编码色值替换为令牌，扩大 `style-guard` 守卫范围。
5. **P2**：沉淀 `SummaryStatGrid / KeyEventTimeline` 等通用组件，形成组件库文档。

> 以上均不触及麻醉记录单本体，可与记录单后续迭代并行推进。
